なぜCamelなのか？

Enterprise Integration Patternsに基づく

Camelは、Gregor Hohpe氏とBobby Woolf氏の素晴らしい本に出てくるEnterprise Integration Patternsのほとんどをサポートしています。また、マイクロサービスアーキテクチャからの新しい統合パターンもサポートしており、ベストプラクティスをすぐに適用してインテグレーションの問題を解決することができます。

EIPの詳細はこちら

多様な実行環境をサポート

Apache Camelはスタンドアロンで、Spring Boot、Quarkus、アプリケーションサーバ、クラウドにライブラリとして組み込むことができます。Camelのサブプロジェクトは、あなたの作業を容易にすることに重点を置いています。

Camelのサブプロジェクトへ

多数の接続コンポーネント

データベース、メッセージキュー、APIなど、基本的にあらゆるものにアクセスするために使用される数百のコンポーネントが含まれています。あらゆるものとの統合をサポートします。

コンポーネントリファレンスへ

50種類以上のデータフォーマットに対応

Camelは約50種類のデータフォーマットをサポートしており、金融、通信、医療などの業界標準フォーマットをサポートしているので、複数のフォーマットのメッセージを変換することができます。
